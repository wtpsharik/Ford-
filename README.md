# Ford Auto Sales
> проект по курсу «Программная инженерия».  

## 📌 Содержание репозитория

- **`/backend`** — Django проект с REST API (модели: Car, Customer, Order, комплектации и т.д.)
- **`/frontend`** — React приложение (Vite) с адаптивным интерфейсом для менеджеров и клиентов
- **`/docs/architecture`** — диаграммы UML (вариантов использования, классов, последовательностей) и текстовые описания архитектуры
- **`/docs/kanban.md`** — доска задач с распределением ролей и статусами
- **`/docs/wiki`** — база знаний: описание выполненных задач, прототипы API, форматы данных
- **`/docs/integration`** — правила интеграции для зависимых задач (прототипы методов, форматы параметров, примеры заглушек)
- **`RULES.md`** — правила оформления кода, работы с Git, роли в команде
- **`TODO.md`** — текущие задачи с приоритетами

##  Организация работы

- Каждый участник выполняет роли **разработчика** (пишет код + структурные тесты) и **тестировщика** (пишет функциональные тесты + проверяет вручную).
- Интеграция компонентов проводится на занятиях. Демонстрация преподавателю — каждую пару.
- Три презентации заказчику (владелец автосалона) за семестр.

##  Функциональные возможности

- **Учёт автомобилей Ford** (модель, год, комплектация, цена, наличие)
- **Управление клиентами** (регистрация, история покупок)
- **Оформление заказов** (резервация, договор купли-продажи)
- **Генерация отчётов** (продажи за период, популярные модели)
- **Поиск и фильтрация** автомобилей по параметрам
- **Административная панель** для сотрудников салона

### Установка бэкенда
cd backend
python -m venv venv
source venv/bin/activate  # или .\venv\Scripts\activate на Windows
pip install -r requirements.txt
cp .env.example .env      # настройте переменные окружения
python manage.py migrate
python manage.py createsuperuser  # для доступа в админку
